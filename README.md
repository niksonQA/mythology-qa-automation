# 🚀 Mythology QA Automation Project

Проект по автоматизации тестирования API для игрового модуля персонажей (Mythology). Фреймворк покрывает тестами полный жизненный цикл сущностей: создание, частичное обновление данных, удаление, механику рекрутинга персонажей и интеграцию с сервисом рейтингов. В проекте реализована сервисная архитектура и настроен полноценный CI/CD пайплайн.

## 🛠 Стек технологий
* **Язык программирования:** Python 3.10+
* **Тестовый фреймворк:** Pytest (fixtures, parametrization)
* **Тестирование API:** Requests
* **Валидация данных:** Pydantic (проверка контрактов JSON-схем)
* **Генерация данных:** Faker
* **Отчетность:** Allure Reports
* **Инфраструктура:** GitHub Actions & Telegram Bot Notifications

---

## 🏗 Архитектура проекта

Фреймворк спроектирован с разделением логики тестов и логики отправки запросов (паттерн API Client / Сервисный слой):

* `api/` — содержит класс `APIClient`, изолирующий HTTP-методы (`GET`, `POST`, `PATCH`, `DELETE`) и эндпоинты бэкенда.
* `tests/` — слой тест-кейсов, использующий клиент через фикстуры.
* `conftest.py` — централизованное управление фикстурами (авторизационная сессия, подключение к БД, автоматическое создание и удаление тестовых персонажей).
